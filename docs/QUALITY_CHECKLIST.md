# Checklist de Qualidade — FinançasPro BI

Este checklist organiza melhorias para consolidar o FinançasPro BI como projeto de dashboard financeiro/PWA.

## 1. Funcionalidades

- [ ] Cadastro de receitas funcionando.
- [ ] Cadastro de despesas funcionando.
- [ ] Seleção de período funcionando.
- [ ] Categorias financeiras funcionando.
- [ ] Previsão de receita funcionando.
- [ ] Indicadores atualizando corretamente.
- [ ] Importação CSV validada.
- [ ] Barra de progresso exibindo dados coerentes.

## 2. Interface

- [ ] Layout mobile validado.
- [ ] Layout desktop validado.
- [ ] Botões com boa área de toque.
- [ ] Formulários com labels claros.
- [ ] Cards alinhados.
- [ ] Estados vazios tratados.
- [ ] Cores com bom contraste.

## 3. PWA

- [ ] Manifest configurado.
- [ ] Service Worker registrado.
- [ ] Ícone local funcionando.
- [ ] Aplicação instalável em ambiente HTTPS.
- [ ] Cache básico funcionando.

## 4. Qualidade técnica

- [ ] Sem erros no console.
- [ ] JavaScript organizado.
- [ ] CSS organizado.
- [ ] Dados tratados antes de cálculo.
- [ ] Validação mínima nos inputs.
- [ ] Código comentado onde houver regra de negócio.

## 5. Documentação

- [ ] README atualizado.
- [ ] `CASE_TECNICO.md` criado.
- [ ] `QUALITY_CHECKLIST.md` criado.
- [ ] Prints adicionados.
- [ ] Link de demo publicado.

## 6. Melhorias futuras

- [ ] Persistência local mais robusta.
- [ ] Exportação PDF.
- [ ] Gráficos por categoria.
- [ ] Backend Node.js.
- [ ] Autenticação.
- [ ] Multiusuário.

## Como apresentar em entrevista

Este projeto deve ser apresentado como dashboard financeiro/PWA que demonstra fundamentos de front-end, lógica de negócio, manipulação de dados, responsividade e experiência de usuário.
